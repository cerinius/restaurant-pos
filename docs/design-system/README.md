# Restaurant POS Design System

This design system is based on the frontend audit in [../ui-ux-audit.md](../ui-ux-audit.md) and is intended to guide incremental UI standardization across the existing codebase.

Documents:
- [design-principles.md](./design-principles.md)
- [foundations.md](./foundations.md)
- [shared-components-priority.md](./shared-components-priority.md)

Goals:
- fast
- obvious
- touch-friendly
- dense but clean
- consistent
- operationally efficient
- easy to scan during busy restaurant service

Constraints:
- preserve the current architecture where possible
- favor semantic naming over one-off page styling
- prefer consistency over novelty
- evolve the existing dark operational language already present in POS, host, KDS, and tenant admin
